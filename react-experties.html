20+ Common Client / Interview Questions & Smart Answers

  • 1. Tell me about your React Native experience.
    I have 3 years of experience in React Native, building cross-platform apps for Android and iOS. I work with APIs, state management, navigation, push notifications, and performance optimization. I focus on delivering high-quality apps on time and collaborating closely with clients.
  • 2. Which state management libraries have you used?
    I’ve used Redux and Context API. Redux for complex apps with large states, Context API for simpler apps where lightweight state management is enough.
  • 3. How do you handle API integration?
    I use Axios or fetch for API calls, handle loading states and errors properly, and structure calls in a separate service layer to maintain clean code.
  • 4. How do you optimize React Native app performance?
    I use FlatList for large lists, React.memo to avoid unnecessary re-renders, optimize images, use lazy loading, and profile components using React DevTools.
  • 5. How do you handle navigation?
    I mainly use React Navigation, organizing Stack, Tab, and Drawer navigators. For complex apps, I manage deep linking and authentication flow efficiently.
  • 6. How do you handle push notifications?
    I use Firebase Cloud Messaging (FCM), handling both foreground and background notifications, and navigate users to the appropriate screens on tap.
  • 7. How do you debug React Native apps?
    I use React Native Debugger, Chrome DevTools, and Flipper for inspecting network requests, layouts, and performance issues.
  • 8. How do you manage app releases?
    For iOS, I use Xcode and TestFlight. For Android, I use Android Studio and Gradle builds. I ensure proper signing, versioning, and compliance with app store guidelines.
  • 9. How do you handle different screen sizes and device orientations?
    I use responsive designs with Flexbox, percentage-based widths/heights, and libraries like react-native-responsive-screen or the Dimensions API to adjust layouts dynamically.
  • 10. How do you manage offline data?
    I use AsyncStorage, SQLite, or Realm to store offline data, and implement sync mechanisms when the app reconnects to the internet.
  • 11. How do you ensure app security?
    I secure API calls using HTTPS, encrypt sensitive data in storage, use authentication tokens securely, and follow best practices to prevent common vulnerabilities.
  • 12. How do you handle animations in React Native?
    I use the Animated API, Reanimated, and Lottie for smooth animations and interactive UI effects.
  • 13. How do you handle forms in React Native?
    I use controlled components and libraries like Formik or React Hook Form, along with Yup for validation.
  • 14. How do you handle real-time updates in apps?
    I use WebSockets, Firebase Realtime Database, or Firestore for real-time data syncing, depending on the project requirement.
  • 15. How do you test React Native apps?
    I write unit tests using Jest, component tests with React Native Testing Library, and sometimes end-to-end tests with Detox.
  • 16. How do you handle deep linking?
    I use React Navigation’s deep linking configuration to handle URL schemes and navigate users to specific screens in the app.
  • 17. How do you handle third-party libraries?
    I carefully choose well-maintained and popular libraries, check compatibility with React Native version, and always test thoroughly before integration.
  • 18. How do you handle performance issues in FlatList or large lists?
    I use keyExtractor, memoized components, getItemLayout, and pagination/lazy loading to maintain smooth performance in large lists.
  • 19. How do you handle memory leaks?
    I clean up subscriptions, timers, and listeners properly in useEffect cleanup functions or componentWillUnmount to prevent memory leaks.
  • 20. How do you handle multiple environments (dev, staging, production)?
    I use `.env` files and react-native-config to manage multiple environments and ensure proper configuration for each build.
  • 21. How do you handle app updates and version control?
    I follow semantic versioning, use Git for source control, and for over-the-air updates, I use CodePush or App Center if needed.
  • 22. How do you ensure cross-platform consistency?
    I test apps thoroughly on both Android and iOS devices, handle platform-specific differences using the Platform module, and use consistent styling and components.

10 Advanced & Tricky React Native Questions & Smart Answers

  • 1. What will you do if your app becomes slow after adding many screens and features?
    I start with profiling using Flipper and React DevTools to identify the bottleneck—whether it's unnecessary re-renders, heavy lists, or large API data. I optimize components with memoization, move heavy processing off the main thread, use FlatList correctly, and ensure minimal computation inside render functions.
  • 2. How do you handle performance when rendering thousands of list items?
    For large data lists, I use FlatList with windowSize, initialNumToRender, removeClippedSubviews, getItemLayout, and memoized card components. I also implement pagination or infinite scrolling so the app never loads all data at once.
  • 3. How do you reduce the app bundle size?
    I remove unused libraries, use ProGuard/Minify for Android, enable Hermes engine, optimize and compress images, use vector icons instead of heavy PNGs, and ensure no duplicate dependencies exist.
  • 4. What if the app crashes only on production but not in development?
    I check crash logs using services like Firebase Crashlytics or Sentry. Production crashes are often due to minification, missing permissions, or build configuration issues. I replicate the production environment locally using release builds and debug step-by-step.
  • 5. Explain the difference between useCallback, useMemo, and React.memo.
    useCallback → memoizes functions. useMemo → memoizes values. React.memo → memoizes components. I use these to prevent unnecessary renders, especially when passing props to child components.
  • 6. How do you handle app-level errors and crashes gracefully?
    I use an error boundary to catch UI crashes, integrate Crashlytics/Sentry for runtime crashes, validate API responses, and use global error handlers. This helps avoid white screens or app freezes.
  • 7. How do you handle native module issues or when a library doesn’t work as expected?
    I check compatibility with the RN version, re-link libraries if autolinking fails, review iOS Pod dependencies, clean build folders, and if required, debug the native code (Swift/Java/Kotlin) to fix the issue.
  • 8. How do you manage sensitive data like tokens?
    I never store sensitive data in AsyncStorage. Instead, I use encrypted storage, secure keychain/KeyStore, HTTPS communication, and rotate tokens securely through backend APIs.
  • 9. When do you choose a native module over a JS-based solution?
    I use native modules when I need access to device-specific features such as Bluetooth, background services, sensors, or when performance-critical tasks need native speed.
  • 10. What is the benefit of using Hermes in React Native?
    Hermes improves startup time, reduces memory usage, speeds up JS execution, and decreases app bundle size. It is especially useful for low-end Android devices.
